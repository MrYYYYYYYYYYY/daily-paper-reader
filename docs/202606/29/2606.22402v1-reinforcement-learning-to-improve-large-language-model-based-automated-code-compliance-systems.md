## 摘要
基于大语言模型（LLM）的建筑规范自动合规（ACC）方法容易生成错误且带有幻觉的计算机可处理规则。本文提出P4IR框架，这是一个两阶段框架，首先使用监督微调（SFT）向LLM注入领域知识，随后通过组相对策略优化（GRPO）提高生成的高级代码骨架形式的中级表示的准确性。与SFT基线相比，该框架在树编辑距离和词级别的莱文斯坦距离上分别降低了23.8%和38.6%。对比分析表明，这种方法在零样本设置下，在代码结构和语义上均优于领先的LLM，具体包括Claude Opus和Sonnet 4.5、GPT-5.2、Qwen-3-Max和GLM-4.7（通过少样本提示评估）。此外，GRPO阶段产生了虽小但统计上显著的误报率降低。通过将SFT与GRPO相结合以直接优化特定领域目标，该方法为构建更准确可靠的基于LLM的ACC系统提供了一条路径。

## Abstract
Large language model (LLM)-based approaches for automated code compliance (ACC) of building regulations are prone to generating incorrect and hallucinated computer-processable rules. This paper introduces P4IR, a two-stage framework that uses supervised fine-tuning (SFT) to instill domain knowledge in an LLM, followed by Group Relative Policy Optimization (GRPO) to improve the accuracy of the generated intermediate representations in the form of high-level code skeletons. The framework achieved reductions of up to 23.8% and 38.6% in tree edit distance and token-level Levenshtein distance respectively, relative to the SFT baselines. Comparative analysis demonstrates that this approach in a zero-shot setting outperforms leading LLMs in both code structure and semantics, specifically Claude Opus and Sonnet 4.5, GPT-5.2, Qwen-3-Max, and GLM-4.7, evaluated via few-shot prompting. Additionally, the GRPO stage produced a small yet statistically significant reduction in false positives. By combining SFT with GRPO to optimize directly for domain-specific objectives, this approach offers a path toward more accurate and reliable LLM-based ACC systems.

## 2. 方法论：核心思想、关键技术细节与算法流程

- **核心思想**：先用SFT注入领域知识（使LLM理解建筑法规与代码结构的映射），再通过GRPO强化学习直接优化代码骨架的结构准确性和句法有效性。GRPO无需critic模型，通过组内输出相对优势来更新策略，计算和内存需求更低。
- **关键组件**：
  - **中间表示（IR）**：采用“代码骨架”（code skeleton）——只包含类/函数定义及其层级关系（名称、参数、嵌套），忽略底层实现细节（替换为`pass`），降低生成难度并专注结构。
  - **奖励函数系统**（图3）：

    - **长度惩罚**（Rlength）：若输出过短（长度低于阈值且与参考长度差超阈值）则奖励-3，防止奖励破解。
  - **GRPO目标函数**（公式3）：包含策略比率的裁剪（clip）和KL散度正则化，确保更新稳定且不偏离SFT知识。

  1. （左半）**SFT阶段**：在预训练LLM（Mistral 7B Instruct v0.3）上使用664个法规-代码骨架对进行监督微调，最小化交叉熵损失，得到初始策略`π_init`。

## 6. 主要结论与发现

1. **GRPO能显著提升SFT性能**：在最优SFT epoch（8-10）上，GRPO使树编辑距离降低23.8%（28.38→21.63），词级Levenshtein距离降低38.6%（602.44→369.93）。
2. **SFT成熟度至关重要**：欠拟合（epoch<8）导致GRPO修复有限；过拟合（epoch≥12）导致策略坍塌（熵迅速归零），GRPO几乎无改进。存在一个最优SFT窗口。
3. **GRPO减少幻觉（假阳性）**：平均假阳性函数数从21.60降至18.60，标准差从23.87降至18.00，统计显著（p=0.005），但效应量小（Cohen’s d=0.127）。
4. **零样本P4IR超越少样本SOTA**：P4IR在所有指标上（树编辑距离20.49，Levenshtein 372.66，Jaccard距离0.7081，CodeBERTScore 0.84）均优于所有基线模型（Claude Opus、GPT-5.2等），且更接近人类专家标注水平（Jaccard距离~0.63）。
5. **语义知识保持稳定**：CodeBERTScore在GRPO过程中几乎不变，说明GRPO主要优化结构而不改变语义。
6. **logit熵可作为GRPO适用性的启发式指标**：适中的熵（epoch 8-10）对应最好性能。

## 7. 优点：方法与实验设计亮点

- **方法论亮点**：
  - 两阶段框架（SFT+GRPO）明确分离“知识注入”和“结构优化”，解决了SFT无法直接优化结构的问题。
  - 奖励函数设计精细：Jaccard相似度捕捉层级签名，语法/长度惩罚防止奖励破解和非法输出。
  - 去除显式推理token，避免领域偏移下推理错误，同时降低计算开销。
  - 使用GRPO替代PPO/RLHF，消除critic模型，减少内存和训练不稳定性。
- **实验设计亮点**：
  - 系统消融SFT epoch与GRPO的交互，并通过logit熵分析揭示模型行为机制（欠拟合-合适窗口-过拟合）。
  - 统计显著性检验验证假阳性减少（p<0.001），增加结论可信度。
  - 定性案例展示具体改进（如修复层级错误、恢复缺失类、消除生成循环）。

## 8. 不足与局限
