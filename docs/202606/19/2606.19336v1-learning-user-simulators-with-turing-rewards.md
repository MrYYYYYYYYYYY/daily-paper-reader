## 摘要
在交互式环境中学习模拟人类用户，可以推动智能助手训练、个性化系统评估、社会科学研究等的发展。现有方法通常通过训练大型语言模型（LLM）来匹配单一的真实响应，要么通过最大化对数概率，要么使用相似性奖励。我们转而提出{Turing-RL}：一种基于图灵测试的强化学习方法，用于训练用户模拟器模型。{Turing-RL}使用判别性图灵奖励，并利用LLM裁判对生成的响应与真实用户基于其历史记录的响应之间的不可区分性进行评分，用户模拟器LLM通过此类奖励学习生成与用户可能说出的内容无法区分的响应。在两个不同的领域——对话聊天和Reddit论坛讨论中——我们发现{Turing-RL}在LLM和人类评估指标上均持续优于基线方法。我们的研究表明，优化不可区分性而非响应匹配，对于学习用户模拟器是有效的。

## Abstract
Learning to simulate human users in interactive settings could advance the training of agent assistants, evaluation of personalization systems, research in the social sciences, and more. Existing approaches generally do so by training a large language model (LLM) to match a single ground truth response, either by maximizing the log probability or by using a similarity reward. We instead propose {Turing-RL}: a Turing-Test-based reinforcement learning approach for training user simulator models. {Turing-RL} uses a discriminative Turing reward with an LLM judge to score how indistinguishable a generated response is from the real user's given the user's history, and the user simulator LLM learns to produce responses indistinguishable from what the user could have said with such rewards. Across two different domains--conversational chat and Reddit forum discussion--we find that {Turing-RL} consistently outperforms baseline methods on both LLM and human evaluation metrics. Our study suggests that optimizing for indistinguishability, rather than response matching, is effective for learning user simulators.

## 1. 论文的核心问题与整体含义

*   **研究动机：** 用户模拟器（User Simulator）在许多领域具有重要价值，例如训练AI助手、评估个性化系统和进行社会科学研究。然而，模拟一个真实个体是一项艰巨的任务。现有方法通常通过让大语言模型（LLM）匹配一个单一的“真实”用户回复（Ground Truth Response）来训练，具体方式为最大化该回复的对数概率或使用相似度奖励。
*   **核心问题：** 这种“匹配式”训练信号忽略了用户行为的本质多样性：同一用户在相同上下文中也可能说出多种不同的话。因此，追求与单个真实回复的重合度，并不能训练出真正“类人”的、能够产生合理多样化回复的模拟器。
*   **整体含义：** 论文提出，应该放弃与特定真实回复进行匹配的策略，转而训练模型生成“与真实用户难以区分”的回复。这一定义更贴近图灵测试的核心思想，即追求不可区分性（Indistinguishability）。

## 2. 论文提出的方法论：Turing-RL

*   **核心思想：** 采用基于图灵测试（Turing Test）的强化学习方法来训练用户模拟器。核心创新在于使用一个**判别性图灵奖励（Discriminative Turing Reward）**：训练一个LLM裁判来判断模型生成的回复与真实用户的回复哪个更“像”该用户，并将这个判断分数作为强化学习的奖励信号。
*   **关键技术细节：**
    1.  **问题定义：** 给定当前对话语境 `x` 和用户表示 `u`（由用户历史行为 `h` 和从历史中归纳出的角色 `ρ` 组成），模型需要生成一个合理的回复 `y`。
    2.  **奖励函数：** 一个LLM裁判（Judge）被展示用户历史、当前语境以及两个回复（一个来自真实用户 `y⋆`，一个来自模型 `y`）。裁判对“哪个回复更像是真人写的”进行1-7分的评估（7分代表模型回复更真）。之后，将评分归一化到 `[0, 1]` 区间，并设置上限（得分>5时强制设为5）以防止奖励黑客（Reward Hacking）。公式为：
       
       `rturing(y, y⋆) = (min{s(y, y⋆), 5} - 1) / 6`
    3.  **训练算法：** 使用**群组相对策略优化（GRPO）** 进行强化学习。对于每个训练样本，模型生成多个候选回复，计算每个回复的奖励，然后在组内进行归一化计算优势值，并更新模型参数。
    4.  **基础模型与预训练：** 使用 Qwen3-8B 作为基础模型。在强化学习之前，先进行**监督微调（SFT）** 作为预热。SFT阶段使用带有“思维链（CoT）”推理轨迹的真实回复进行训练。
*   **流程简述：** SFT初始化策略 → 对给定用户历史生成多个候选回复 → 使用LLM裁判（Turing Judge）评估每个回复的不可区分性 → 根据评估的奖励分数，使用GRPO算法优化策略，使其能生成更不可区分的回复。

## 3. 实验设计

*   **数据集/场景：**
    1.  **对话聊天（Chat）：** 使用 **PRISM Alignment Dataset**，包含多轮人机对话，涉及1,500名参与者。
    2.  **网络论坛讨论（Reddit）：** 使用 **ConvoKit** 子版块语料库，包含Reddit上的多层讨论。
*   **基准方法（Baselines）：**
    1.  **Sim-RL：** 使用**响应相似度奖励（Response Similarity Reward）**，衡量生成回复与真实回复的内容匹配度（改编自Wu et al., HumanLM）。
    2.  **Logprob-RL：** 使用**对数概率奖励（Log-probability Reward）**，最大化真实回复在模型下的条件对数概率（改编自Gandhi et al., 2026）。
    3.  **SFT-Init：** 仅完成监督微调，未进行强化学习的模型。
    4.  另外，还对比了未训练的模型如GPT-5、Qwen3.5-397B等作为参考。

        *   **图灵可区分性（Turing Distinguishability）：** 核心指标，衡量回复有多像真人。
        *   **响应相似度（Response Similarity）：** 衡量与真实回复内容的重合度。
        *   **上下文与用户特异性（Context and User Specificity）：** 衡量回复是否与当前交互紧密相关且符合目标用户特点。
