## 摘要
在线自蒸馏（OPSD）通过让模型基于自身 rollout 进行训练，并使用冻结拷贝提供以参考目标为条件的密集词元级目标。该方法在 LLM 推理中效果良好，但直接扩展到多模态大语言模型（MLLM）可能产生捷径：特权目标可能主要基于文本参考目标而非图像来引导词元。我们提出 ViGOS，一种用于 MLLM 后训练的视觉引导在线自蒸馏框架。学生模型首先撰写视觉描述，然后推理出最终答案。对于有效 rollout，仅基于图像的感知教师监督描述，而特权推理教师则在相同学生前缀上监督推理和最终答案。对于无效 rollout，仅使用参考教师恢复输出格式。在通用视觉-语言、专家推理、视觉数学、空间定位和视觉-语言先验基准测试中，ViGOS 保留了 OPSD 的主要优势，并在易出现捷径的环境中提升了基于图像的行为。

## Abstract
On-policy self-distillation (OPSD) trains a model on its own rollouts and uses a frozen copy to provide dense token-level targets conditioned on a reference target. This works well for LLM reasoning, but a direct extension to multimodal large language models (MLLMs) can create a shortcut: the privileged target may guide tokens mainly based on the text reference target rather than the image. We propose ViGOS, a visually grounded OPSD framework for MLLM post-training. The student first writes a visual description and then reasons toward the final answer. For valid rollouts, an image-only perception teacher supervises the description, while a privileged reasoning teacher supervises the reasoning and final answer on the same student prefix. A reference teacher is used only for invalid rollouts to recover the output format. Across general vision-language, expert reasoning, visual math, spatial grounding, and visual-language-prior benchmarks, ViGOS keeps the main benefits of OPSD and improves image-grounded behavior in shortcut-prone settings.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：在线策略自蒸馏（On-Policy Self-Distillation，OPSD）在纯文本大语言模型（LLM）推理中表现优异，通过让模型基于自身的 rollout 进行训练，并使用冻结的拷贝提供以参考目标为条件的密集词元级目标。
- **问题**：直接将 OPSD 扩展到多模态大语言模型（MLLM）时，会产生一种危险的**捷径（shortcut）**：特权教师（可看到参考答案）在监督整个 rollout 时，主要依赖文本参考目标而非图像来引导词元，导致学生模型学习到答案兼容但视觉依据薄弱的推理。这类似于 VQA 中的语言先验和捷径学习问题。
- **核心意义**：需要一种方法，在保留 OPSD 密集监督优势的同时，防止文本参考目标过早干预视觉感知，激励模型真正基于图像内容进行推理。

## 2. 方法论：核心思想、关键技术细节与算法流程

- **核心思想**：将推理过程解耦为“感知（视觉描述）”和“推理（基于描述的答案推断）”两个阶段，对不同阶段使用不同监督信号，从而控制特权信息（参考答案）进入的时机。
- **关键技术细节**：
  - **结构化输出**：学生模型必须生成 `<description> d </description> <think> r </think> \boxed{a}` 格式的响应，其中 d 是视觉描述，r 是推理过程，a 是最终答案。
  - **三种教师角色**（均来自学生冻结副本，仅外部上下文不同）：
    1. **图像仅感知教师（q_img）**：仅接受图像 I 作为外部上下文，**监督描述部分** `d` 的生成。计算损失 `L_perc`（KL 散度）。
    2. **特权推理教师（q_rea）**：接受图像 I、问题 x 和参考答案 a*，**监督推理和答案部分** (r, a)。计算损失 `L_rea`。
    3. **参考教师（q_ref）**：接受与 q_rea 相同的特权信息，但**仅用于无效 rollout**（格式错误的输出），作为回退信号使用逆向 KL 散度 `L_ref`。
  - **rollout 有效性判断**：通过解析器检查是否格式正确、描述和推理非空、能解析答案。有效时启用 q_img 和 q_rea；无效时启用 q_ref。
- **算法流程**（文字描述）：
  1.  学生模型在给定图像和问题下，采样一个结构化轨迹 y = (d, r, a)。
  2.  解析器提取词元掩码 Td, Tr, Ta，并判断 rollout 是否有效（minv）。
  3.  **若有效**：对描述词元 Td，计算与图像仅感知教师 q_img 的 KL 散度损失 L_perc；对推理+答案词元 Tr∪Ta，计算与特权推理教师 q_rea 的 KL 散度损失 L_rea。
  4.  **若无效**：对所有词元 Ty，计算与学生分布 p_θ 与参考教师 q_ref 的逆向 KL 散度损失 L_ref。
  5.  总损失 `L = λ_perc L_perc + λ_rea L_rea + λ_ref L_ref`。
- **关键公式**：感知损失 `L_perc = E[(1-minv(y)) ΣD_KL(q_img,t ∥ p_θ,t)]`；推理损失 `L_rea = E[(1-minv(y)) ΣD_KL(q_rea,t ∥ p_θ,t)]`；回退损失 `L_ref = E[minv(y) ΣD_KL(p_θ,t ∥ q_ref,t)]`。所有教师均基于学生当前前缀 ht 进行条件化，保持 on-policy 特性。

## 8. 不足与局限

- **实验覆盖的局限**：虽然 benchmark 丰富，但主要基于 Qwen2.5-VL 系列 backbone，在更大尺度（如 72B/13B）或其他架构（如 LLaVA、InternVL）上的泛化性尚未验证。
- **方法本身的局限**（论文也提及）：
  - 生成的描述可能不完整或不正确；图像仅感知教师可能产生通用描述。
  - 训练需要额外的教师前向传播（每个 rollout 需调用3趟蒸馏），计算开销略高。
  - 复杂符号推理和响应稳定性方面，ViGOS 的 Avg@5 在某些任务上仍低于 Baseline（如 7B 的 MathVerse、MathVista 的 Avg@5 略有下降），表明在纯符号推理任务上解耦可能不是最优。
- **偏差风险**：ViGOS 鼓励先描述后推理，可能对需要快速凭直觉回答的问题（如简单 VQA）引入不必要的过细化描述，降低效率。
- **应用限制**：方法要求模型在生成时严格遵循结构化格式，但训练时如果存在大量格式违例，参考教师回退信号可能仍会引入一些特权信息。此外，该方法尚未与 RLHF 等后训练方法结合评估。
