## 摘要
随着企业部署基于检索增强生成（RAG）的系统来为用户查询提供基于事实的响应，重排序已成为最终过滤步骤的关键组件，用于将相关文档与干扰或不相关文档区分开。现有的重排序器通常依赖启发式阈值来实现最优过滤。此外，对于相关性评分，最先进的方法使用语言模型的logit信号，这些信号是为下一个词元预测设计的，而非用于评估相关性。为解决这些局限性，我们识别了一个原则性的相关性信号：当以文档为条件时，查询内部状态中引发的表征变化（RS）。我们观察到，（a）候选文档引发的RS与（b）Oracle文档集引发的RS之间的对齐程度，提供了相关性的稳健指标。基于这一洞察，我们引入了一个轻量级训练框架，学习将RS映射到校准的相关性得分的投影。我们的训练目标自然地以零阈值过滤不相关内容，减少了对启发式调优的依赖。在多种检索数据集上，我们的方法相较于最先进的重排序器取得了性能提升。

## Abstract
As enterprises deploy RAG-based systems to provide grounded responses to user queries, reranking has become a critical component for the final filtering step that separates relevant from distracting or irrelevant documents. Existing rerankers often rely on heuristic thresholds to achieve optimal filtering. Moreover, for relevance scoring, state-of-the-art methods use a language model's logit signals, which are designed for next-token prediction, not for assessing relevance. To address these limitations, we identify a principled signal for relevance: the representational shift (RS) induced in a query's internal state when conditioned on a document. We observe that the alignment between (a) RS induced by a candidate document and (b) RS induced by an oracle document-set provides a robust indicator of relevance. Building on this insight, we introduce a lightweight training framework that learns projections mapping RS to calibrated relevance scores. Our training objectives naturally filter irrelevant content at a zero threshold, reducing dependence on heuristic tuning. Across diverse retrieval datasets, our method delivers gains over SOTA rerankers.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：现有基于大语言模型（LLM）的重排序器在RAG系统中广泛使用，但存在两个关键问题：(1) **阈值不一致**：不同数据集、甚至同一数据集的不同查询，最优阈值变化很大，导致固定 top-k 或固定阈值策略效果差；(2) **相关性信号不匹配**：当前方法利用语言模型的 logit 或下一词元预测信号来评估相关性，这些信号并非为相关性判别设计，导致分数缺乏校准。
- **整体含义**：本文提出一种**基于内部表征位移**的新信号——**Representational Shift (RS)**，即当查询被文档“条件化”时，其内部状态（value 向量）的变化。通过将 RS 投影到校准后的分数空间，可以自然地实现零阈值区分相关/不相关文档，从而减少对启发式调优的依赖，提升重排序的可靠性和跨数据集泛化能力。

### 2. 论文提出的方法论：核心思想、关键技术细节、公式 / 算法流程

- **核心思想**：利用 Transformer 的 value 向量变化来量化文档对查询的影响。  
  - **Oracle RS**：将所有相关文档同时输入模型得到的 RS，代表“理想”的相关性方向。  
  - **假设**：若一个文档的 RS 与 Oracle RS 在几何上对齐，则该文档更可能相关。
- **关键技术细节**：
  - **表征位移定义**：对于每个 query token 在特定注意力头 (ℓ, h) 处的 value 向量，文档 \(d\) 引起的位移为  
    \[
    \delta v_t^{(\ell,h)}(d;q) = v_{n+t}^{(\ell,h)}(d \cdot q) - v_{n+t}^{(\ell,h)}(q)
    \]  
    利用 RoPE 特性，将 null 前缀替换为仅 query 的基线，消除位置干扰。
  - **RS 张量**：收集所有选定的层、头、query token 位置上的位移，形成张量 \(\Delta(d,q)\)。
  - **投影学习**：训练一个轻量投影矩阵 \(B \in \mathbb{R}^{L \times H \times P \times D}\)（P=64，总参数2.3M），将 RS 投影到 P 维空间。然后计算投影向量与全1向量 \(\mathbf{1}_P\) 的余弦相似度作为分数：
    \[
    s(d|q) = \sum_{\ell,h} \frac{1}{T}\sum_t \cos(B_{\ell,h}\,\Delta(d,q)_{\ell,h,t},\mathbf{1}_P)
    \]
  - **训练损失**：包含五个项——校准损失（推相关>0、不相关<0）、间隔损失（强制间隔 m=0.5）、正交正则化、Oracle 对齐损失、Frobenius 范数约束。总损失为：
    \[
    \mathcal{L} = \lambda_1\mathcal{L}_{cal} + \mathcal{L}_{margin} + \lambda_2\mathcal{L}_{ortho} + \lambda_3\mathcal{L}_{align} + \lambda_4\mathcal{L}_{frob}
    \]
- **算法流程**（文字描述）：
  1. 对每个 query，计算仅 query 的 value 向量作为基线。
  2. 对每个候选文档，计算 query+doc 的 value 向量，减去基线得到 RS 张量。
  3. 将 RS 通过学习的投影 B 映射到 P 维，计算余弦相似度并求和得分数。
  4. 推理时直接使用零阈值（\(s > 0\) 为相关）进行过滤。

### 6. 论文的主要结论与发现

- **RSRank 优于 SOTA**：平均 NDCG@5 87.3（+1.0 pp vs Qwen3-Reranker-8B），Recall@5 81.1（+2.0 pp），特别是在 2WikiMQA、MuSiQue 上提升明显。
- **自然阈值校准**：F1@τ 从 Qwen3-Reranker-8B 的 60.3 提升至 67.5，且零阈值跨数据集稳定（Bias 降低 17 倍，Variance 降低 47 倍）。
- **样本高效**：仅需 50 个样本即可达到接近满配性能（R@5 89.2 vs 800 样本 93.9）。
- **轻量高效**：冻结 LLM + 2.3M 参数 + 极低推理开销，适合生产部署。
- **RS 空间具有可分离性**：通过学习投影，相关/不相关文档的 RS 在投影后形成清晰聚类（UMAP 可视化）。

### 7. 优点

- **理论简洁**：利用有限差分思想，借助 RoPE 特性简化基线，数学优雅。
- **训练高效**：冻结 LLM，仅训练 2.3M 投影参数，数据需求小（2000 样本），避免了全模型微调的高成本。
- **阈值自然化**：训练目标直接生成以零为决策边界的校准分数，免去人工调参，跨数据集稳定。
- **实验全面**：包括性能对比、阈值分析、消融、样本效率、可视化，以及错误分析（F1 分解）。
- **实用性**：推理开销极小（仅增加 0.88% 时间），符合生产级 RAG 场景要求。

### 8. 不足与局限

- **负样本选择偏差**：在 BEIR 数据集（FiQA、FEVER、NFCorpus）中使用随机负样本而非 hard negatives（如 BM25 或密集检索第一阶段的返回结果），可能高估绝对性能。论文自己也承认这一点。
- **查询级校准仍有改善空间**：RSRank 虽然解决了数据集级校准，但查询级校准依然不够完美（如 HotpotQA 上仍有 F1 差距 10.3pp），需要未来工作探索动态阈值。
- **长文档泛化**：实验仅针对段落/句子级 chunk，未验证对整篇文章的冗长文档的重排序效果，适用范围受限。
- **对 backbone 的依赖**：RS 本身依赖于特定 LLM 的内部表征（value 向量），更换 backbone 可能需要重新提取 RS 或微调投影，迁移性有待验证。
- **多个训练损失项带来调参复杂性**：虽然有消融证明各组件有效，但 λ 超参数（λ1~λ4）仍需少量调整，对从业者有一定门槛。
- **未与基于 LLM 的 listwise/setwise 方法对比**：论文主要对比 pointwise reranker，并因效率问题未直接比较 listwise（如 RankZephyr）或 setwise（如 TourRank），这些方法在候选集较小时可能有更高精度。
