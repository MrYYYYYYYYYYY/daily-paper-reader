## 摘要
在翻译之前使用大型语言模型（LLM）重写源文本已被证明能提高机器翻译（MT）质量。然而，我们发现基于提示的重写反而可能降低翻译质量，尤其是在使用较小LLM（如4B参数模型）时。我们认为，这一限制源于仅通过自然语言提示难以控制重写行为：只有当重写能改善下游翻译时才是有用的，但现有的基于提示的方法并未明确优化这一信号。为解决此问题，我们提出RLSR（用于源文本重写的强化学习），这是一个强化学习框架，它根据每次重写产生的下游翻译质量改进来训练重写模型。跨六个MT系统和16个语言对的实验表明，我们使用4B参数的RLSR训练的重写模型在相同模型规模下显著优于无重写基线和基于提示的重写基线，同时与使用235B LLM的基线保持竞争力。

## Abstract
Rewriting source text with large language models (LLMs) before translation has been shown to improve machine translation (MT) quality. However, we find that prompt-based rewriting can degrade translation quality rather than improve it, particularly when smaller LLMs, such as 4B-parameter models, are used. We argue that this limitation stems from the difficulty of controlling rewriting behavior through natural-language prompts alone: a rewrite is useful only if it improves downstream translation, yet existing prompt-based methods do not explicitly optimize for this signal. To address this issue, we propose RLSR (Reinforcement Learning for Source Rewriting), a reinforcement learning framework that trains the rewriting model with a reward based on the downstream translation-quality improvement produced by each rewrite. Experiments across six MT systems and 16 language pairs show that our 4B RLSR-trained rewriting models significantly outperform both the no-rewriting baseline and prompt-based rewriting baselines at the same model scale, while remaining competitive with baselines that use a 235B LLM.

# 论文总结：Rewrite to Translate, Translate to Reward: Reinforcement Learning for Source Rewriting in Machine Translation

## 1. 核心问题与整体含义（研究动机和背景）
- **研究问题**：在机器翻译（MT）中，使用大型语言模型（LLM）对源文本进行重写（预编辑）可提升翻译质量，但基于自然语言提示的重写方法存在不可靠性——尤其当使用较小模型（如4B参数）时，重写反而可能降低翻译质量。
- **核心原因**：自然语言提示只能间接表达意图（如“简化文本”），而无法直接优化“下游翻译质量提升”这一关键信号；模型存在“提示跟随能力”的脆弱性，小模型更容易产生有害重写。
- **研究目标**：提出一种直接优化下游翻译质量的源重写方法，使小模型也能获得可靠、显著的翻译质量改善。

## 2. 方法论
### 核心思想
- 将源重写建模为**强化学习（RL）**问题：重写模型作为策略，奖励定义为重写前后下游MT模型翻译质量的**提升值**（即重写后翻译得分减去原源翻译得分）。通过RL直接优化该奖励，替代传统的基于提示的间接控制。

### 关键技术细节
- **奖励定义**：  
  R(s, ˜s, r) = Q(s, M(˜s), r) − Q(s, M(s), r)  
  其中s为原源，˜s为重写源，M为固定MT模型，Q为自动评估指标（如xCOMET-XL），r为参考译文。
- **目标函数**：  
  最大化期望调整后奖励：  
  J(θ) = E_{s∼D, ˜s∼R_θ(·|s)} [R(s, ˜s, r) − β log(R_θ(˜s|s)/R_ref(˜s|s))]  
  其中β控制KL散度惩罚，防止奖励黑客行为（如生成无意义文本利用指标漏洞）。
- **训练算法**：采用**DAPO**（Decoupled Clip and Dynamic Sampling Policy Optimization），一种在线策略RL算法，优于Group Relative Policy Optimization（GRPO）。
- **框架流程**：  
  1. 初始重写模型为参考策略（如给定提示的LLM）。  
  2. 从当前策略采样重写候选。  
  3. 固定MT模型翻译原源和重写源。  
  4. 计算奖励并优化策略参数。

## 3. 实验设计
### 数据集
- **训练数据**：来自WMT2019–2024新闻/通用MT共享任务测试集，共21K样本，覆盖24个语言对。随机取5%作为验证集，其余用于训练。
- **测试数据**：完整WMT2025通用MT共享任务测试集，共**16个语言对**（涵盖英语、捷克语、日语三种源语言）。

### 基准（Benchmark）
- **MT模型（下游）**：6个，包括通用LLM（Gemma3 27B、Gemma4 31B、Qwen3 30B、Qwen3 4B）和翻译优化模型（Translategemma 27B、自研Qwen3 4B SFT-RL）。
- **重写模型**：Qwen3 4B和Qwen3 235B（仅用于提示基线，RLSR仅训练4B模型）。
- **对比方法**：
  - **无重写（No-rewriting）**：直接翻译原源。
  - **基于提示的重写**：采用Ki & Carpuat (2025)三种提示策略（Simplification、Paraphrase、Easy Translate），分别使用4B和235B LLM。
  - **RLSR**（本文方法）：使用Easy Translate提示初始化，训练4B模型。
- **评价指标**：xCOMET-XXL（学习型参考指标）、MetricX-24-XXL（学习型参考指标）、GEMBA-MQM（基于Gemini 3.0 Flash的提示型无参考指标）。

## 6. 主要结论与发现
1. **基于提示的重写在小模型上不可靠**：所有Qwen3 4B提示基线在xCOMET上均低于无重写基线，部分情况下退化严重。
2. **RLSR显著优于无重写和同规模提示基线**：在大多数设置下统计显著（p<0.05），跨源语言稳健。
3. **RLSR 4B模型与235B大模型竞争力相当**：无需大模型即可达到相近翻译质量，提供参数高效替代方案。
4. **RL优于SFT**：RL训练的重写模型避免了SFT导致的“复制退化”（大量输出与原源相同），生成更多样、更有针对性的重写。
5. **跨MT模型泛化良好**：为某一MT模型训练的RLSR可有效迁移至另一MT模型，联合训练也能兼顾多个模型。
6. **RLSR学习局部、长度保持的编辑**：不盲目全量重写，而是针对实际翻译障碍（如不流畅、歧义）进行精准修改。

## 7. 优点
- **方法创新**：首次将RL直接用于源重写，以翻译质量改进为奖励，克服了提示控制的间接性和不可靠性。
- **实验设计严谨**：覆盖6个MT模型、16个语言对、3个评价指标，并进行统计检验；额外分析SFT/DPO对比、跨模型泛化、定性案例，验证方法有效性。
- **结果显著**：4B小模型达到与235B大模型竞争力相当的性能，实用价值高。
- **透明度高**：公开讨论了指标偏差风险，并用独立指标验证；明确报告算力消耗和限制。

## 8. 不足与局限
1. **指标偏差风险**：RLSR使用xCOMET作为奖励，可能过度拟合学习型指标。尽管GEMBA验证缓解了此顾虑，但最严谨的做法仍应是人工评估（因成本未做）。
2. **实验范围有限**：仅覆盖6个MT模型（虽来自不同家族），但未涵盖更多样化的模型架构；语言对虽多但仅含三种源语言（英、捷、日），缺乏低资源语言或领域。
3. **训练成本高**：RLSR训练约60 GPU小时/模型，每适配一个新MT模型原则上需重新训练，成本限制了可扩展性。
4. **跨模型泛化仅验证了两模型**：虽观察到良好迁移性，但仅测试了两个MT模型（Gemma3 27B和Qwen3 30B），更大规模验证缺失。
5. **未与更强基线对比**：未包含Koretaka等人（2025）的迭代推理重写方法（因其代码未公开），可能遗漏更先进的提示增强方法。
6. **SFT改进方向未深入**：论文指出SFT可通过上采样关键标记改进，但未实施该方案，仅止于讨论。
