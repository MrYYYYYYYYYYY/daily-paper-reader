## 摘要
强化学习（RL）已成为提升大型语言模型（LLM）在复杂推理任务上表现的核心方法，然而RL后训练在很大程度上与之前的预训练孤立研究。因此，两个基本问题仍然悬而未决：（1）预训练选择（模型大小、数据）如何影响RL计算回报，以及（2）RL实际上对模型做了什么？在标准的LLM设置中研究这些问题很困难：预训练语料库庞大且不受控制，难以将行为归因于预训练与RL，并且跨两个阶段的系统计算扫描成本过高。为了解决这些挑战，我们使用国际象棋作为受控测试平台，研究从预训练到后训练完整流水线中的推理。我们遵循标准LLM训练流程：在人类国际象棋对局上预训练从5M到1B参数的语言模型，在合成推理轨迹上进行监督微调（SFT），并在可验证奖励的国际象棋谜题上运行RL。利用这一框架，我们发现给定RL计算水平下的后RL表现可以从预训练损失很好地预测，并且RL奖励曲线的斜率随预训练令牌数近似线性改善。除了规模效应，我们发现RL并非简单地锐化SFT策略：在简单谜题上，它放大了SFT策略已偏好的正确走法；在困难谜题上，它凸显了在SFT下几乎不存在的正确走法。我们进一步测试了这些发现是否超越国际象棋：在数学领域文本上训练了一个1B语言模型，出现了相同的预测模式——预训练时间更长的检查点在RL下达到更高的后RL性能，并且改进更快。总之，我们提供了预训练到RL接口的定量描述，以及一个用于研究从预训练到后训练完整流水线中推理科学的受控测试平台。

## Abstract
Reinforcement learning (RL) has become central to improving large language models (LLMs) on complex reasoning tasks, yet RL post-training is largely studied in isolation from the pretraining that precedes it. As a result, two basic questions remain open: (1) how do pretraining choices (model size, data) shape the returns to RL compute, and (2) what does RL actually do to the model? These questions are difficult to study in the standard LLM setting: pretraining corpora are vast and uncontrolled, making it hard to attribute behaviors to pretraining versus RL, and systematic compute sweeps across both stages are prohibitively expensive. To address these challenges, we use chess as a controlled testbed for studying reasoning across the full pretraining-to-post-training pipeline. We follow the standard LLM training pipeline by pretraining language models from 5M to 1B parameters on human chess games, supervised fine-tuning on synthetic reasoning traces, and running RL on chess puzzles with verifiable rewards. Using this framework, we find that the post-RL performance at given RL compute level is well-predicted from the pretraining loss, and slope of the RL reward curves improves approximately linearly with the pretraining tokens. Beyond scaling, we find that RL does not simply sharpen the SFT policy: on easy puzzles it amplifies correct moves the SFT policy already preferred, while on hard puzzles it surfaces correct moves that were nearly absent under SFT. We further test whether our findings transfer beyond chess by training a 1B language model on math-domain text, where the same predictive pattern emerges: longer-pretrained checkpoints reach higher post-RL performance and improve faster under RL. In sum, we provide a quantitative account of the pretraining-to-RL interface and a controlled testbed for studying the science of reasoning across the full pretraining-to-post-training pipeline.

# 理解从预训练到后训练中的推理 (Understanding Reasoning from Pretraining to Post-Training)

## 一、核心问题与研究动机

大型语言模型（LLM）的标准训练流程包括大规模**预训练**（Pretraining）和后训练（Post-training，通常包含监督微调 SFT 和强化学习 RL）。尽管 RL 在提升 LLM 复杂推理能力方面越来越重要，但现有研究往往**孤立看待预训练和 RL 后训练**，导致两个基本问题仍未解决：

1. **预训练选择如何影响 RL 的回报？** 即模型大小、数据量等因素如何改变 RL 计算的有效性。
2. **RL 实际上对模型做了什么？** 它仅仅是锐化已有能力，还是能创造新能力？

在自然语言场景中研究这些问题面临三个主要障碍：
- 预训练语料库庞大且异构，难以将行为归因于预训练还是 RL；
- 跨两个阶段的系统计算扫描成本过高；
- 评估通常只提供最终答案正确性，对每一步推理行为的分析不透明。

因此，论文选择**国际象棋（Chess）**作为受控测试平台，构建一个镜像标准 LLM 训练流水线的框架，以量化预训练与 RL 的交互。

## 二、方法论：核心思想与技术细节

### 整体框架

论文构建了一个三阶段流水线模型，镜像标准 LLM 训练：

1. **预训练阶段**：在人类国际象棋对局数据上，训练自回归策略模型（next-token prediction）。
2. **监督微调（SFT）阶段**：使用**合成推理轨迹**在谜题上训练，模型学习在做出最终走法前生成思考过程。
3. **强化学习（RL）阶段**：基于可验证奖励的谜题环境，使用 GRPO 算法优化策略。

### 核心技术细节

- **棋步表示**：每个走法用四个 token 编码：`<piece> <source> <destination> <flag>`，词汇表大小为 81。
- **合成推理轨迹生成**：给定起始局面，从提议策略采样多条可能的游戏延续，将它们合并为前缀树，以深度优先顺序序列化为推理轨迹（`<T> ... <sep> ... </T>`）。
- **RL 奖励**：二进制奖励，模型必须在完整解线中每一步都正确才能获得奖励 1，否则为 0。
- **RL 算法**：采用 Group Relative Policy Optimization (GRPO)。

### 联合预训练-RL 缩放定律

论文建立了一个局部对数线性模型来拟合 RL 学习曲线：

```
R_{N,T}(C) = Rref_{N,T} + B_{N,T} (log10 C - log10 Cref)
```

其中：
- `Rref_{N,T}` 是给定参考 RL 计算量 `Cref` 下的奖励水平，可通过预训练验证损失预测。
- `B_{N,T}` 是局部 RL 斜率，表示每增加一个数量级的 RL 计算带来的性能增益，可通过预训练 token 数和模型尺寸预测。

最终联合定律可以预测任意假设训练配方 `(N, T, C_RL)` 的后 RL 性能。

- 主要评估指标：**pass@1**（单次采样正确率）、**pass@16**（16 次采样内至少一次正确）。

- 论文不与其他现有方法直接竞争，而是聚焦于理解预训练与 RL 的交互。
- 在 SFT 阶段，对比了有无推理轨迹的微调效果，证明了推理轨迹对 pass@k 指标的必要性。

## 六、主要结论与发现

### 1. 联合预训练-RL 缩放定律
- **预训练损失预测后 RL 性能**：给定 RL 计算量下的 pass@1 性能与预训练验证损失高度相关（Spearman 相关系数从 -0.93 到 -0.99），且随 RL 计算增加而增强。
- **预训练数据量预测 RL 学习速度**：RL 性能每十年计算提升的斜率 `B_{N,T}` 随预训练 token 数近似线性增长（Spearman +0.84，Pearson +0.84）。
- 联合模型（log T + log N）解释斜率达 R²=0.84，其中 token 数贡献大于模型尺寸。

### 2. 计算分配策略
- **pass@1 视角**：在固定总计算预算下，最优 RL 分配比例随总计算量增加而增大（从 20% 到 28%），表明在较高计算预算下 RL 应获得更大比例。
- **pass@16 视角**：RL 提升有限，更长的预训练往往更有效。这与 Yue et al. (2025) 的发现一致。

### 3. RL 作用的机制
- **RL 不是简单的均匀锐化**：实际效果因谜题难度而异。
  - **简单谜题 (B1-B2)**：RL 主要放大 SFT 策略已偏好的正确走法（ground-truth amplification）。
  - **困难谜题 (B4-B5)**：RL 同时出现两种效果：**尾部发现**（激活 SFT 几乎不存在的正确走法）和**错误放大**（强化错误走法）。
- 这些异质性解释了为什么 RL 能改善 pass@1 但不一致改善 pass@16：概率质量以多种方式重新分配。

### 4. 跨域可迁移性
- 在 1B 语言模型的数学域实验中，**相同的预测模式成立**：更长的预训练导致更高的后 RL 性能和更陡的局部 RL 斜率。

3. **联合缩放定律**：首次提供了预训练与 RL 之间的定量接口描述，填补了孤立研究的空白。
4. **细致的机理分析**：通过策略演化分析，揭示了 RL 在不同难度任务上不同的作用模式，为 RL 改进提供了具体方向（如减少错误放大）。
5. **跨域验证**：在数学域上的实验证明了发现超出国际象棋场景的潜力。
6. **开源贡献**：提供了模型、数据集和代码，便于复现和扩展。

## 八、不足与局限

1. **域间差异限制**：国际象棋与自然语言的推理本质不同——词汇表极小 (81 token)、验证精确、不涉及世界知识和流畅性。发现的缩放参数不应直接迁移到语言模型。
2. **RL 环境限制**：使用具有唯一确定解线的谜题和二进制奖励，是受限的验证形式，与自然语言任务中的部分学分或开放奖励不同。
3. **模型规模有限**：最大 1B 参数，超出范围的缩放行为未知，尤其是在前沿模型规模。
4. **计算预算限制**：RL 运行未达到饱和阶段，局部线性拟合接近实际但非完整 sigmoid 曲线，限制了渐进性分析的可靠性。
5. **拟合局限性**：
   - 斜率 `B_{N,T}` 的估计噪声较大，比 `Rref` 更难预测；
   - 严格外推（仅用 N, T）比使用观察到的预训练损失（观察损失）误差大 50%；
   - 当前参数化不包含优化器细节、数据关注或其他运行级变量。
6. **推理轨迹格式固定**：仅使用特定树序列化格式，不同选择可能产生不同 RL 下 CoT 演化动态。
7. **数学域验证有限**：仅使用 1B 模型，14 个检查点，数据范围较小，属于定性案例研究而非严格缩放验证。
